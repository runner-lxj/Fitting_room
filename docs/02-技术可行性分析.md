# 智能试衣间 — 技术可行性分析

> 版本: v0.1 | 更新日期: 2026-07-02 | 状态: 初稿

## 1. 技术约束条件

| 约束项           | 说明                                                          |
| ---------------- | ------------------------------------------------------------- |
| 前端             | 微信小程序（WXML + WXSS + JS），不使用 UniApp/Taro            |
| 业务后端         | 微信云开发（云函数 + 云数据库 + 云存储），无自建服务器        |
| AI 推理          | 需要外部 GPU 服务器（微信云函数不支持 GPU，内存上限 256MB）   |
| 预算             | MVP 阶段零预算，全部使用开源模型和免费额度                    |
| 模型部署         | 开源模型自部署，选用轻量级、推理速度快的方案                  |

---

## 2. 架构总览

微信小程序 (前端) -> 微信云开发 (业务层) -> GPU 推理服务器 (AI 层)

GPU 服务包含：数字人生成、衣服抠图、虚拟试穿、推荐引擎。

关键决策：微信云函数无法运行 AI 模型，必须部署外部 GPU 服务器。
MVP 阶段可用个人电脑的 NVIDIA 显卡（GTX 1060 6GB 以上），后期迁移到云 GPU。

---

## 3. 各模块技术选型

### 3.1 数字人生成（照片级真实 + 多角度 2D 切片）

推荐方案：Stable Diffusion + ControlNet + IP-Adapter

| 方案                      | 效果 | 成本   | 速度        | 适用性 |
| ------------------------- | ---- | ------ | ----------- | ------ |
| SD + ControlNet + IP-Adapter | 优秀 | GPU 推理 | 10-20s/张 | 推荐 |
| InstantID                 | 优秀 | GPU 推理 | 15-30s/张 | 可选   |
| Ready Player Me API       | 良好 | API 付费 | 3-5s       | 不推荐 |
| 简单贴图叠加              | 差   | 零     | < 1s        | 不推荐 |

技术路线：
1. 用户上传正面照片
2. InsightFace 提取人脸特征
3. IP-Adapter 将人脸特征注入 SD 模型
4. ControlNet（OpenPose）控制人体姿态
5. 生成不同姿态全身照：正面、3/4 侧面、背面
6. GFPGAN 做面部增强

预估性能：
- 首次生成全套（3-5 张）：60-120 秒
- 模型占用显存：约 6-8 GB
- 推荐显卡：RTX 3060 12GB 或更高

### 3.2 衣服抠图 + 摆正

推荐方案：rembg（基于 U2-Net / RMBG-2.0）

| 方案                 | 效果 | 成本 | 速度     |
| -------------------- | ---- | ---- | -------- |
| rembg (U2-Net/RMBG) | 优秀 | 免费 | 0.5-1s   |
| SAM (Segment Anything)| 优秀 | 免费 | 2-4s     |
| remove.bg API        | 优秀 | 付费 | 0.3s     |

理由：开源免费，调用简单，GPU 加速后 < 0.5 秒。

摆正处理：OpenCV 轮廓检测 + 最小外接矩形 + 仿射变换校正，耗时 < 0.1 秒。

### 3.3 AI 虚拟试穿（核心难点）

推荐方案：IDM-VTON（首选）+ CatVTON（备选/降级）

| 方案         | 效果 | 显存需求 | 推理速度  | 开源 |
| ------------ | ---- | -------- | --------- | ---- |
| IDM-VTON     | 优秀 | 6-8 GB   | 8-12s     | 是   |
| OOTDiffusion | 优秀 | 8-10 GB  | 10-15s    | 是   |
| HR-VITON     | 良好 | 4-6 GB   | 5-8s      | 是   |
| CatVTON      | 良好 | 2-4 GB   | 3-5s      | 是   |

IDM-VTON 是目前 VTON-Human 基准上效果最好的开源模型之一：
- 生成质量：IDRiD 分数最高
- 支持上衣/下装/全身试穿
- 支持衣物细节保留（纹理、褶皱）
- GitHub 2k+ stars，社区维护活跃

CatVTON 作为轻量级备选（显存低、速度快）。

潜在风险与应对：
1. 试穿出现伪影/变形 -> 调整 CFG scale；提供「重新生成」按钮
2. 衣物与身体比例不匹配 -> 生成前根据身高体重缩放衣物图
3. 非正面视角效果差 -> MVP 仅支持正面；v1 迭代多角度

### 3.4 穿搭推荐引擎

方案：规则引擎（零成本）+ LLM 辅助（免费额度）

规则层温度-服装映射：
| 温度区间    | 推荐类别       |
| ----------- | -------------- |
| > 28C       | 短袖/短裤/裙子 |
| 22-28C      | T恤/薄长裤     |
| 15-22C      | 卫衣/牛仔裤    |
| 8-15C       | 夹克/厚长裤    |
| < 8C        | 羽绒服/厚外套  |

LLM 层：MiMo-v2.5（已有额度），生成搭配理由文案。
每日 1-3 次/用户，免费额度充裕。

### 3.5 美颜处理

方案：OpenCV 基础美颜（MVP 阶段足够）
- 轻度：双边滤波（磨皮）+ 颜色空间调整（美白）
- 中度：+ 面部关键点检测 + 局部增强
- 重度：接入 GFPGAN 面部增强

## 4. GPU 服务器需求

模型显存占用估算：
| 模型             | 显存占用 |
| ---------------- | -------- |
| SD + ControlNet  | ~4 GB    |
| IDM-VTON         | ~4 GB    |
| rembg            | ~0.5 GB  |
| 合计             | ~8.5 GB  |

结论：12GB 显存显卡可同时加载所有模型。8GB 需做模型切换，延迟增加 5-10 秒。

MVP 阶段硬件：
- 个人 PC，NVIDIA GTX 1060 6GB 以上
- 或租用云 GPU（AutoDL / 矩池云，约 1-3 元/小时）
- 内存 16GB+，硬盘 50GB+

---

## 5. 微信云开发适配

能做的（云函数）：
- 用户数据 CRUD（衣柜、个人信息、搭配历史）
- 天气 API 调用
- MiMo-v2.5 推理调用
- 文件上传/下载管理
- 定时任务触发（凌晨预生成）

不能做的（需 GPU 服务器）：
- 图像生成/推理（数字人、试穿、抠图）

云函数调用 GPU 服务器流程：
1. 用户触发试穿 -> 云函数接收请求
2. 云函数从云存储获取用户照片 + 衣物图
3. 云函数调用 GPU 服务器 HTTP API（传入图片 URL/二进制）
4. GPU 服务器推理，返回结果图
5. 云函数将结果上传云存储，返回给小程序

延迟预估：
| 操作             | 总耗时    | 构成                      |
| ---------------- | --------- | ------------------------- |
| 数字人生成       | 60-120s   | SD 推理 5 张 x 15s       |
| 衣服入库         | 3-5s      | 抠图 1s + 分类 0.5s      |
| 单件试穿         | 8-12s     | IDM-VTON 推理            |
| 组合试穿（3件）  | 20-30s    | 3 次 IDM-VTON 叠加推理   |
| 穿搭推荐（含图） | 10-15s    | 规则匹配 + IDM-VTON      |

## 7. 风险评估

| 风险                   | 严重性 | 可能性 | 应对措施                             |
| ---------------------- | ------ | ------ | ------------------------------------ |
| 试穿效果不够真实       | 高     | 中     | IDM-VTON 是 SOTA 模型；MVP 优先验证 |
| GPU 服务器成本超预期   | 中     | 低     | 自建 PC 零成本；云 GPU 按需计费     |
| 数字人生成质量不稳定   | 中     | 中     | 限制输入照片质量；提供重试           |
| 微信云函数超时         | 低     | 低     | 云函数仅做中转，AI 推理在外部       |
| 衣服抠图不精准         | 低     | 低     | rembg 已很好；可切换 SAM            |
| 微信审核不通过         | 中     | 中     | 避免敏感内容；用户协议明确           |

## 9. 结论

技术可行性结论：可行，但有明确边界。

1. AI 试穿：IDM-VTON 效果达到 SOTA 水平，开源免费，推理速度可接受（8-12s）。这是 MVP 最核心的验证点，技术上可行。
2. 数字人生成：SD + ControlNet + IP-Adapter 方案成熟，可生成多角度 2D 切片。12GB 显存显卡可胜任。
3. 衣服处理：rembg 抠图效果优秀，OpenCV 摆正简单可靠。无技术风险。
4. 穿搭推荐：规则引擎零成本，LLM 有免费额度。无技术风险。
5. 架构约束：微信云开发 + 外部 GPU 服务器的架构完全可行。

最大风险在于试穿效果是否达到用户期望。建议优先搭建最小化试穿 Demo（数字人底图 + IDM-VTON），验证核心效果后再推进全量开发。
