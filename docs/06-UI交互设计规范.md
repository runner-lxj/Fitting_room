## 3. 页面设计

### 3.1 首页（index）

**布局结构：**
1. **问候区**（顶部 8rpx 内边距）
   - 左侧：问候语 + 穿搭建议 + 城市定位胶囊
   - 右侧：数字人形象（280rpx 圆形，叠在天气卡右上角）
2. **天气卡片**
   - 温度大字（44px）+ 天气描述 + 风力湿度
   - 右侧圆形天气图标
   - 底部穿搭建议
3. **今日推荐**
   - 标题行 + "换一批"按钮
   - 单套搭配卡片居中显示
4. **快捷入口**（底部，贴 TabBar）
   - 我的衣柜 / 添加衣服 / 我的穿搭

**交互：**
- 页面切换: fade + slide（300ms）
- "换一批": 循环切换推荐方案
- 快捷入口点击跳转对应页面

---

### 3.2 衣柜页（wardrobe）

**布局结构：**
1. 标题区：「我的衣柜」+ 总数
2. 分类 Tab（胶囊形，可横滑）：全部/上衣/下装/连衣裙/外套/鞋子/配饰
3. 二级分类筛选（可选）
4. 双列网格：衣服卡片（图片区 300rpx + 标签+颜色圆点）
5. 右下角 FAB：+ 添加衣服

**交互：**
- Tab 切换：胶囊变色 + 数据筛选
- 衣服卡片：点击查看详情，长按删除
- FAB 点击：跳转添加衣服页
- 搭配计数标签：衣服被采纳后显示"搭1套啦~"

---

### 3.3 我的穿搭页（my-outfits）

**布局结构：**
1. 标题区 + 一键搭配胶囊按钮（标题右侧）
2. Tab 切换：待采纳 / 已采纳
3. 搭配列表：出装图缩略图（144x144rpx）+ 信息 + 箭头
4. 右上角浮窗（生成中/完成状态）

**一键搭配按钮状态逻辑：**

| 条件 | 按钮文案 | 状态 |
|------|---------|------|
| pendingCount === 0 | "一键搭配" | 可点击 |
| pendingCount > 0 | "再来一批" | 可点击 |
| generating === true | "生成中..." + spinner | disabled |

**浮窗状态机：**

`
空闲 → [点击一键搭配] → 生成中 → [完成] → 完成
                                              ↓ [点击浮窗]
                                              空闲
`

| 状态 | 浮窗标题 | 进度条 | 状态文字 | 按钮 |
|------|---------|--------|---------|------|
| 空闲 | 隐藏 | — | — | — |
| 生成中 | "搭配生成中" | 暖杏渐变，随进度增长 | 轮流显示阶段文字 | 无 |
| 完成 | "搭配已生成" | 满格变绿 | "N 套方案就绪" | "查看方案" |

**浮窗样式：**
- 位置: 固定右上角，导航栏下方
- 尺寸: 280rpx 宽
- 样式: 毛玻璃白底（blur 16px）+ 12rpx 圆角
- 动画: 从上方滑入（translateY -16rpx → 0，300ms ease）

### 3.4 搭配详情页（outfit-detail）

**显示规则：**

| match_count | 显示 | 样式 |
|-------------|------|------|
| 0 | 无角标 | - |
| 1 | 已搭配1套 | 暖杏色半透明底 |
| 2 | 已搭配2套 | 深色半透明底（满配） |

**数据来源：**
- 查询 outfits 集合 WHERE status=accepted AND items 包含该衣服 ID
- COUNT 作为 match_count
- 仅统计已采纳方案，pending 的不算
- 上限 2 套

**WXML 示例：**
`html
<view class="match-badge {{item.match_count >= 2 ? 'full' : ''}}" wx:if="{{item.match_count > 0}}">
  <text>已搭配{{item.match_count}}套</text>
</view>
`

**JS 数据计算（loadClothes 后）：**
`js
const acceptedOutfits = await db.collection('outfits')
  .where({ user_id: '{openid}', status: 'accepted' }).get()
const matchMap = {}
acceptedOutfits.data.forEach(o => {
  (o.items || []).forEach(i => {
    const id = i.id || i
    matchMap[id] = (matchMap[id] || 0) + 1
  })
})
clothes.forEach(c => {
  c.match_count = Math.min(matchMap[c._id] || 0, 2)
  c.match_tag = c.match_count > 0 ? '已搭配' + c.match_count + '套' : ''
})
`

## 5. 交互流程

### 5.1 首次使用流程

`
打开小程序
  → 微信授权登录
  → 首页（空衣柜状态）
  → 引导上传人身照 → AI 分析体型风格
  → 引导拍照添加衣服 → 识别分类入库
  → 衣柜有衣服后 → 点击一键搭配
`

### 5.2 一键搭配流程

`
用户在穿搭页点击「一键搭配」
  → 按钮变为 loading 状态（spinner + "生成中..."，disabled）
  → 右上角弹出浮窗，显示进度
  → 调用云函数 outfit-recommend → MiMo-v2.5 生成搭配
  → 调用云函数 outfit-compose → Pillow 拼接出装图
  → 完成后浮窗变为"搭配已生成" + "查看方案"
  → 用户点击浮窗 → 跳转待采纳列表
  → 查看搭配详情 → 采纳/舍弃
  → 采纳后衣服 match_count +1
  → 按钮文案切换（有方案→"再来一批"，无方案→"一键搭配"）
`

### 5.3 衣服入库流程

`
点击 FAB「+」
  → wx.chooseMedia 选择图片（最多9张）
  → 图片立刻显示在列表（状态：上传中）
  → 异步上传到云存储
  → 写入数据库（status: pending）
  → 调用 clothes-add 云函数识别
  → 识别完成 → 更新数据库（status: recognized）
  → 用户可点击编辑识别结果
  → 点击"完成"返回衣柜
`

### 5.4 搭配采纳流程

`
在穿搭页点击搭配卡片
  → 进入搭配详情页
  → 查看：出装图 + AI搭配理由 + 单品列表
  → 点击「采纳」
  → 更新 outfits.status = 'accepted'
  → 更新衣服 match_count
  → 返回穿搭页（已采纳 tab）
`
