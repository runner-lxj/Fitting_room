# 智能试衣间 — 产品需求文档 (PRD)

> 版本: v0.1 | 更新日期: 2026-07-02 | 状态: 初稿

## 1. 产品概述

### 1.1 产品愿景
一款微信小程序，帮助用户管理个人衣橱并通过 AI 技术实现虚拟试穿，解决「买了很多衣服但不知道怎么搭配」的痛点。

### 1.2 目标用户
主力人群：18-35 岁女性，有网购习惯，衣橱中 20-50+ 件单品，有搭配困扰但预算有限。

### 1.3 核心价值主张
- 建立个人数字衣橱，拍照即入库
- AI 驱动的虚拟试穿，所见即所得
- 基于天气和场景的智能搭配推荐

### 1.4 MVP 验证假设
**核心假设：AI 虚拟试穿效果足够好，用户愿意使用并复购/分享。**
MVP 阶段聚焦于此假设验证，其他功能为辅助验证服务。

---

## 2. 功能模块详细描述

### 2.1 模块一：用户注册与数字人生成

**功能描述**
用户首次登录后，通过上传正面照片 + 填写身高体重，AI 生成照片级真实的数字人形象。

**用户流程**
1. 微信授权登录（获取 openid）
2. 引导用户拍摄/上传一张正面清晰照片（拍摄指引：光线充足、背景简洁、正面站立）
3. 用户填写身高、体重（可选：年龄、体型偏好）
4. 后端对照片进行美颜处理（磨皮、美白、亮度调整）
5. 基于美颜后照片 + 身体参数，AI 生成多角度数字人形象（正面/侧面/背面，2D 切片）
6. 用户预览生成结果，满意则确认保存；不满意可重新上传照片

**约束与规则**
- 照片要求：正面、无遮挡、单人、光线充足，尺寸不小于 500x500px
- 美颜程度可调节（轻度/中度/重度），默认中度
- 数字人生成耗时约 15-30 秒，需显示加载动画
- 生成失败时提供重试入口

**数据产出**
- 基础数字人（base_avatar）：无衣物的裸身形象（用于试穿底图）
- 多角度数字人切片（avatar_slices）：正面/3/4 侧面/背面，共 3-5 张

---

### 2.2 模块二：智能衣柜

**功能描述**
用户拍照上传衣服，系统自动完成：抠图去背景 -> 衣物识别分类 -> 摆正整理 -> 存入衣柜。

**用户流程**
1. 点击「添加衣服」，调用相机或相册
2. 拍照/选择图片后，系统自动处理：
   - 精准抠图（去除背景杂质）
   - 衣物分类（上衣/裤子/裙子/外套/鞋子/配饰）
   - 摆正处理（仿射变换纠正倾斜）
3. 用户确认：可编辑分类、添加颜色/品牌/季节标签
4. 保存到个人衣柜

**约束与规则**
- 支持单品拍摄，不支持多件同框
- 处理耗时约 3-5 秒
- 每件衣服需包含：抠图后的透明底图、分类、颜色、可搭配季节
- 图片存储在微信云存储中

**衣柜展示**
- 卡片流布局，按分类 Tab 筛选（全部/上衣/裤子/裙子/外套）
- 每张卡片显示：透明底衣服图 + 分类标签 + 颜色标记
- 支持删除、编辑信息

---

### 2.3 模块三：AI 虚拟试穿

**功能描述**
将衣柜中的衣服「穿」到数字人身上，生成照片级真实的试穿效果图。

**用户流程**
1. 进入试穿页面，数字人形象展示在中央
2. 下方显示衣柜衣服列表，可按分类筛选
3. 点击一件衣服 -> 系统自动将其穿戴到数字人身上 -> 生成试穿图
4. 试穿图可保存到相册、分享给好友
5. 支持连续搭配：点击第二件衣服（如裤子），在已有上衣基础上叠加

**约束与规则**
- 试穿生成耗时约 8-15 秒，需显示进度/加载动画
- 当前 MVP 仅支持正面视角静态图
- 每次试穿生成一张效果图
- 同时穿戴上限：3 件（上衣 + 下装 + 外套/配饰）
- 试穿结果不持久化存储（按需生成），节省存储空间

**进阶功能（v2，MVP 后迭代）**
- 多角度试穿（侧面/背面）
- 试穿结果保存到历史记录
- 搭配方案保存与分享

---

### 2.4 模块四：智能穿搭推荐

**功能描述**
根据用户地址、当天天气、未来天气预报，结合衣柜中已有衣服，给出穿衣建议并一键生成搭配试穿图。

**用户流程**
1. 首页展示今日天气卡片（自动获取位置或用户手动设置）
2. 显示穿搭建议文字：「今天 15度 多云，建议穿卫衣+牛仔裤+薄外套」
3. 点击「一键试穿」-> 自动生成该搭配方案的试穿效果图
4. 提供 2-3 套备选方案，用户可切换

**约束与规则**
- 天气数据来源：和风天气 API（免费版 1000 次/天）
- 穿搭推荐基于规则引擎 + LLM 辅助生成
- 每日预生成：凌晨 2:00-5:00 为用户预生成未来 3 天的穿搭方案（缓存到云数据库）
- 用户手动点击时，优先读取缓存；缓存未命中则实时生成
- 位置权限：用户可选择授权或手动输入城市

**推荐逻辑（分层）**
1. 规则层（快速、无成本）：温度区间 + 场景标签 -> 匹配衣柜中合适的衣服
2. LLM 层（精细、需 API 调用）：调用 MiMo-v2.5 推理模型（已有额度），生成搭配理由和文字建议

---

### 2.5 模块五：个人中心

**功能描述**：用户信息管理与设置。

**功能项**
- 身体数据编辑（身高/体重/体型）
- 数字人重新生成
- 衣服管理（编辑/删除）
- 位置设置
- 关于/反馈入口

## 3. 信息架构

`
小程序
+-- 首页（天气卡片 + 今日推荐 + 快捷入口）
+-- 衣柜（分类 Tab + 衣服卡片流 + 添加按钮）
+-- 试穿（数字人展示 + 衣服选择 + 生成试穿图）
+-- 推荐（搭配方案列表 + 一键试穿）
+-- 我的（个人信息 + 设置）
`

## 4. 页面原型概要

### 4.1 首页
- 顶部：天气卡片（温度/天气/穿搭建议摘要）
- 中部：今日推荐搭配（横向滑动卡片，显示搭配预览）
- 底部：快捷入口（衣柜/试穿/推荐）

### 4.2 衣柜页
- 顶部：分类 Tab 筛选
- 主体：网格卡片，每张卡片 = 透明底衣服图 + 分类标签
- 右下角：悬浮添加按钮

### 4.3 试穿页
- 上半部分：数字人展示区（居中，占据 60% 屏幕高度）
- 下半部分：衣服选择区（横向滚动，分类 Tab）
- 生成按钮：「开始试穿」

### 4.4 推荐页
- 顶部：日期 + 天气信息
- 中部：推荐方案卡片（搭配文字 + 缩略图）
- 点击卡片展开：详细搭配 + 试穿效果图 + 「一键试穿」

## 5. 非功能性需求

| 维度     | 要求                                        |
| -------- | ------------------------------------------- |
| 性能     | 试穿图生成 < 15 秒；衣服处理 < 5 秒；页面加载 < 2 秒 |
| 可用性   | 微信原生体验，操作步骤不超过 3 步到达核心功能 |
| 存储     | 单用户衣柜上限 100 件（MVP 阶段）           |
| 安全     | 用户照片不外传，仅用于生成；数据加密存储     |
| 兼容性   | iOS 12+ / Android 7+，适配主流机型屏幕      |

---

## 6. MVP 范围定义

### MVP 包含（P0）
1. 微信登录 + 数字人生成（正面照片 + 美颜 + 2D 切片）
2. 衣柜管理（拍照 -> 抠图 -> 分类 -> 入库）
3. AI 虚拟试穿（正面静态图，单件/组合试穿）
4. 穿搭推荐（天气规则匹配 + 文字建议）

### v1 包含（P1）
5. 多角度试穿（侧面/背面）
6. LLM 辅助搭配建议
7. 试穿历史与搭配保存

### v2 包含（P2）
8. 社交分享（试穿图分享到朋友圈）
9. 搭配社区（用户发布搭配方案）
10. 视频试穿（动态展示）

## 7. 成功指标

| 指标               | 目标值 | 说明                               |
| ------------------ | ------ | ---------------------------------- |
| AI 试穿好评率      | >= 70% | 用户主动保存/分享试穿图的比例       |
| 衣柜人均入库量     | >= 10  | 用户上传衣服数量                   |
| 日活跃留存率       | >= 30% | 7 日留存                           |
| 单次试穿耗时       | < 15s  | 生成到展示完成                     |
