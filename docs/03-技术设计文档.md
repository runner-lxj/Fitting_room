## 3. 云函数设计

### 3.1 云函数列表

| 云函数名 | 触发方式 | 功能 |
|----------|----------|------|
| user-login | 小程序调用 | 微信登录 + 用户初始化 |
| body-analyze | 小程序调用 | 上传人身照 -> MiMo-v2.5 分析体型风格 |
| clothes-add | 小程序调用 | 上传衣服 -> 抠图 -> 分类 -> 入库 |
| outfit-recommend | 小程序调用/定时触发 | 天气+衣柜 -> MiMo-v2.5 推荐搭配 |
| outfit-compose | 小程序调用 | 搭配方案 -> Pillow 拼接出装图 |
| outfit-accept | 小程序调用 | 采纳搭配 -> 生成衣橱照 -> 存档 |
| notification-list | 小程序调用 | 获取站内信通知列表 |
| notification-read | 小程序调用 | 标记通知已读 |
| admin-config | 小程序调用 | 管理员读取/更新配置 |
| daily-recommend | 定时触发(凌晨) | 批量为活跃用户预生成搭配 |

### 3.2 核心云函数详细设计

#### outfit-recommend（搭配推荐）

`
输入：
  - user_id: 用户 openid
  - trigger: "manual" | "scheduled"

处理流程：
  1. 获取用户 style_profile（体型风格）
  2. 获取用户衣柜 clothes（按分类）
  3. 调用和风天气 API 获取当前天气
  4. 构造 MiMo-v2.5 请求：
     - system: 管理员配置的推荐 prompt
     - user: [
         图片1: 用户人身照,
         图片2-N: 衣柜衣服图片（可选，如果 token 允许）,
         文本: 天气数据 + 衣柜列表 JSON
       ]
     - response_format: json_object
  5. 解析 MiMo-v2.5 返回的搭配 JSON
  6. 校验 item_id 是否存在于衣柜
  7. 保存搭配方案到 outfits 集合
  8. 异步触发出装图拼接
  9. 发送站内信通知

输出：
  - outfits: 搭配方案列表（含出装图 URL）
`

#### outfit-compose（出装图拼接）

`
输入：
  - outfit_id: 搭配方案 ID

处理流程：
  1. 从 outfits 集合获取搭配方案
  2. 从 clothes 集合获取三件衣服的 cutout_url
  3. 下载三张抠图 PNG
  4. Pillow 拼接：
     - 创建 750x1000 白色底图
     - 上衣：居中上方，宽度 60%，y=50
     - 裤子：居中中部，宽度 50%，y=380
     - 鞋子：居中下方，宽度 40%，y=700
  5. 上传到云存储
  6. 更新 outfits 集合的 composition_url

输出：
  - composition_url: 出装图云存储 URL
`

## 4. MiMo-v2.5 集成设计

### 4.1 API 调用封装

`python
# 云函数中调用 MiMo-v2.5
import json
from openai import OpenAI

def call_mimo(messages, config):
    client = OpenAI(
        api_key=config["api_key"],
        base_url="https://api.xiaomimimo.com/v1"
    )
    
    completion = client.chat.completions.create(
        model=config.get("model", "mimo-v2.5"),
        messages=messages,
        response_format={"type": "json_object"},
        temperature=config.get("temperature", 0.7),
        top_p=config.get("top_p", 0.95),
        max_completion_tokens=4096
    )
    
    return json.loads(completion.choices[0].message.content)
`

### 4.2 图片输入方式

**云存储图片 -> MiMo-v2.5 的流程**：
1. 云存储中的图片需要先获取临时访问 URL
2. 将临时 URL 传入 MiMo-v2.5 的 image_url
3. 或者下载图片后转为 Base64 传入

**推荐方案**：使用 Base64 编码（更可靠，不依赖临时 URL 有效期）

`python
import base64

def image_to_base64_url(cloud_file_id):
    # 从云存储下载图片
    file_content = cloud.get_file(cloud_file_id)
    b64 = base64.b64encode(file_content).decode()
    mime = "image/jpeg"  # 根据实际格式
    return f"data:{mime};base64,{b64}"
`

### 4.3 搭配推荐 Prompt 模板

`
系统提示词（管理员可配置）：

你是专业的穿搭推荐顾问，擅长根据用户的体型、风格偏好和天气条件，
从用户的个人衣柜中选择最合适的搭配方案。

用户信息：
- 体型：{body_type}
- 风格偏好：{style_preference}
- 颜色偏好：{color_preference}

当前天气：
- 温度：{temp}°C
- 天气：{condition}
- 风力：{wind}

用户衣柜（JSON 格式）：
{clothes_json}

请推荐 {outfit_count} 套搭配方案，输出 JSON 格式：
{
  "outfits": [
    {
      "name": "方案名称（简短有吸引力）",
      "weather_context": "天气描述",
      "items": {
        "top": {"item_id": "衣柜中的衣服ID", "name": "衣服名称"},
        "bottom": {"item_id": "衣柜中的衣服ID", "name": "衣服名称"},
        "shoes": {"item_id": "衣柜中的衣服ID", "name": "衣服名称"}
      },
      "description": "详细搭配说明（80-120字，说明为什么这样搭、适合什么场合、整体风格）",
      "occasion": "适合场合",
      "score": 0.0-1.0
    }
  ]
}

规则：
1. 每套方案必须包含上衣、裤子、鞋子各一件
2. 只能从用户衣柜中选择已有的衣服
3. 搭配说明要具体，说明颜色搭配、风格协调、场合适配
4. 考虑温度区间的穿衣建议
5. 避免连续推荐相同或过于相似的方案
6. 只输出 JSON，不要其他文字
`

### 6.1 管理员权限控制

function checkAdmin(openid) {

### 6.2 数据安全

- 用户照片仅用于 AI 分析，不外传、不用于其他用途

## 7. 部署方案

### 7.1 MVP 阶段（全云开发）

`
微信小程序
    |
    v
微信云开发
    +-- 云函数（Node.js / Python）
    +-- 云数据库（MongoDB）
    +-- 云存储（图片/文件）
    |
    +---> MiMo-v2.5 API（外网 HTTP）
    +---> 和风天气 API（外网 HTTP）
`

### 7.2 异步处理架构

| 操作 | 前端行为 | 后台处理 | 通信方式 |
|------|----------|----------|----------|
| 衣服上传 | 立刻显示缩略图 | 上传云存储->写库->调用识别 | wx.cloud.callFunction（不阻塞） |
| 衣物识别 | 显示转圈动画 | MiMo-v2.5 API->更新数据库 | 云函数完成后 setData |
| 搭配推荐 | 显示加载动画 | 天气API+MiMo推荐+出图拼接 | Promise await |
| 搭配采纳 | 提示已采纳 | 更新状态->生成衣橱照->写库 | wx.cloud.callFunction |
| 每日预生成 | 无感知 | 定时触发->批量生成->写通知 | 定时触发器 |

### 7.2 扩展阶段（如需独立服务器）

- 抠图服务独立部署（Flask/FastAPI + rembg）
- 使用云函数 HTTP 触发器调用
- 或使用云托管（微信云开发的容器服务）
