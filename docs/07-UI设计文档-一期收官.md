## 1. 项目概述

### 1.1 产品定位
基于天气和个人衣橱，通过 AI 智能推荐每日穿搭方案的微信小程序。

### 1.2 目标用户
18-35 岁有网购习惯的女性用户，衣橱中 20-50+ 件单品，日常有搭配困扰。

### 1.3 一期功能范围

| 模块 | 功能 | 状态 |
|------|------|------|
| 首页 | 天气展示 + 今日推荐 + 快捷入口 | 已完成 |
| 衣柜 | 衣服管理（分类/筛选/删除） | 已完成 |

## 4. 各页面 UI 说明

### 4.1 首页（index）

布局：问候语+数字人头像 → 天气卡片 → 今日推荐(单套) → 快捷入口(3个)

交互：
- 页面加载 → 获取天气 + 从 accepted 方案筛选今日推荐
- "换一批" → 切换推荐方案
- 点击推荐卡片 → 跳转搭配详情
- 快捷入口 → 跳转对应页面

### 4.2 衣柜页（wardrobe）

布局：标题 → 分类Tab(可横滑) → 二级分类 → 双列网格(衣服卡片+角标) → FAB[+]

交互：
- Tab 切换 → 筛选分类
- 长按衣服 → 删除
- FAB → 添加衣服页
- match_count角标 → 来自 accepted outfits 统计

### 4.3 穿搭页（my-outfits）

布局：标题+一键搭配按钮 → 待采纳/已采纳Tab+批量图标 → 搭配列表(网格缩略图) → 浮窗

一键搭配按钮状态：
- pendingCount === 0 → "一键搭配"
- pendingCount > 0 → "再来一批"
- generating → spinner + "生成中"，disabled

交互：
- 点击一键搭配 → 调用云函数 → 浮窗显示进度 → 完成后点击查看
- 批量模式 → 多选后批量采纳
- 点击搭配 → 搭配详情

### 4.4 搭配详情页（outfit-detail）

布局：自定义导航栏(返回) → 出装图网格 → AI理由卡片 → 搭配信息 → 单品列表 → 操作栏

交互：
- "采纳" → 更新 status=accepted + match_count+1
- "舍弃" → 更新 status=discarded → 返回
- 已采纳状态 → 底部显示"已采纳"，无操作

### 4.5 添加衣服页（clothes-add）

布局：标题+计数 → 已上传网格(状态标签+删除) → 上传区域 → 编辑弹窗 → 底部完成

状态流转：uploading → pending → recognizing → recognized/failed

交互：
- 选择图片 → 异步上传+识别
- 点击已识别项 → 编辑分类/颜色/材质
- "完成" → 返回衣柜

### 4.6 个人中心页（profile）

布局：渐变头部(头像+昵称+统计) → 菜单列表 → 关于信息

### 4.7 管理员配置页（admin）

布局：MiMo模型配置(模型/Temperature/TopP/套数) → 系统提示词 → 保存按钮

### 5.1 users
_id: openid | phone, is_admin, body_photo_url, style_profile{}, location

### 5.2 clothes
_id: auto | user_id, name, category, subcategory, color, material, style_tags[]
original_url, cutout_url, status, match_count, match_tag, added_at

### 5.3 outfits
_id: auto | user_id, name, weather{}, items[{id,role}], composition_url
ai_reason, style_summary, weather_tags[], temperature_range{}
occasion, score, status(pending/accepted/discarded), created_at

_id: mimo_config | model, temperature, top_p, outfit_count, system_prompt
