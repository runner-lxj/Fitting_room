# 智能试衣间 — 技术设计文档

> 版本: v0.1 | 更新日期: 2026-07-02 | 状态: 初稿

## 1. 系统架构

### 1.1 整体架构

```
+------------------+     HTTPS      +------------------+     HTTPS      +------------------+
|   微信小程序      | <============> |   微信云开发      | <============> |   GPU 推理服务器  |
|   (WXML/WXSS/JS) |                |   (云函数/云DB)   |                |   (FastAPI/PyTorch)|
+------------------+                +------------------+                +------------------+
        |                                   |                                   |
        | wx.login()                        | HTTP API                          | Model Inference
        | wx.request()                      | MongoDB                           | GPU (CUDA)
        | wx.chooseImage()                  | File Storage                      |
```

### 1.2 技术栈

| 层级     | 技术选型                           | 说明                     |
| -------- | ---------------------------------- | ------------------------ |
| 前端     | 微信小程序原生框架                  | WXML + WXSS + JavaScript |
| 业务后端 | 微信云开发                          | 云函数(Node.js) + 云数据库 + 云存储 |
| AI 推理  | Python + FastAPI + PyTorch          | 自部署 GPU 服务器        |
| AI 模型  | SD/ControlNet/IP-Adapter/IDM-VTON  | 开源模型                 |
| 天气服务 | 和风天气 API                        | 免费版 1000 次/天        |
| LLM      | MiMo-v2.5 API       | 已有额度    |

**POST /api/tryon**

请求体：


**GET /api/tryon/status/{task_id}**

响应体：
```json

  "task_id": "uuid",
  "status": "completed|processing|failed",
  "progress": 1.0,
  "result": {
    "tryon_image_url": "试穿效果图URL",
    "items_applied": ["top", "bottom"]
  }
}
```

#### 2.3.3 处理流程（核心）

1. 云函数接收试穿请求
2. 从云存储获取：用户 base_avatar + 目标衣物 cutout 图
3. 调用 GPU 服务器 IDM-VTON：
   a. 输入：base_avatar（人物图） + 衣物 cutout 图（服装图）
   b. 模型推理：将衣物"穿"到人物身上
   c. 输出：试穿效果图
4. 如果有多件衣物（上衣+裤子）：
   a. 第一次：base_avatar + 上衣 -> 中间结果
   b. 第二次：中间结果 + 裤子 -> 最终结果
5. 返回试穿图 URL

#### 2.3.4 关键参数配置

IDM-VTON 推理参数：
```python
{
  "denoising_strength": 0.4,    # 控制衣物保留度，越小越保留原图细节
  "cfg_scale": 2.0,             # 引导强度
  "num_inference_steps": 30,    # 推理步数，越大质量越好但越慢
  "seed": -1                    # 随机种子，-1 为随机
}
```

衣物缩放逻辑：
- 根据用户身高体重计算缩放比例
- 标准身高参考：165cm -> 1.0x
- 衣物图缩放到与人物比例匹配

#### 2.4.3 推荐生成流程

**实时生成流程：**
1. 获取用户位置 -> 调用和风天气 API 获取天气
2. 规则引擎匹配：
   a. 根据温度区间筛选衣柜中合适的衣服
   b. 按类别分组（上衣、下装、外套）
   c. 生成 2-3 套搭配方案
3. LLM 生成文案：
   a. 将搭配方案 + 天气信息发送给MiMo-v2.5
   b. 生成个性化推荐文案
4. 调用 GPU 服务器为每套方案生成试穿效果图
5. 保存到 outfit_recommendations 集合

**预生成流程（凌晨定时任务）：**
1. 云函数定时触发器：每日凌晨 2:00 触发
2. 遍历活跃用户（7 天内有登录的用户）
3. 获取未来 3 天天气预报
4. 为每个用户生成 3 天 x 3 套 = 9 套搭配方案
5. 预生成试穿效果图
6. 保存到数据库（缓存）

+#### 2.5.4 异步任务机制
+
+AI 推理耗时长（60-120s），不适合同步等待。采用异步任务模式：
+
+1. 客户端发送请求 -> 返回 task_id
+2. 后台线程池异步执行推理
+3. 客户端轮询 status 接口获取结果
+4. 使用 Redis 或内存字典存储任务状态
+
+MVP 阶段使用内存字典即可（单用户并发低）：
+```python
+tasks = {}  # task_id -> {"status": "processing|completed|failed", "result": None, "progress": 0.0}
+```
+
+#### 2.5.5 鉴权方案
+
+GPU 服务器需要验证调用方身份。MVP 阶段使用 API Key：
+- 云函数调用 GPU 服务器时，Header 携带 `Authorization: Bearer {API_KEY}`
+- GPU 服务器验证 API Key 后才执行推理
+- API_KEY 存储在云函数环境变量和 GPU 服务器 config 中
