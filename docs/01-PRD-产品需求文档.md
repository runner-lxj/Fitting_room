# 智能穿搭助手 — 产品需求文档 (PRD)

> 版本: v0.2 | 更新日期: 2026-07-07 | 状态: 新方向定稿

## 1. 产品概述

### 1.1 产品愿景
一款微信小程序，基于天气和用户个人衣橱，通过 AI 智能推荐每日穿搭方案，帮助用户「用好已有的衣服」。

### 1.2 目标用户
18-35 岁有网购习惯的用户，衣橱中 20-50+ 件单品，日常有搭配困扰。

### 1.3 核心价值主张
- 上传一张人身照，AI 分析体型风格
- 拍照入库衣服，建立个人数字衣橱
- 结合实时天气，从衣柜中智能推荐搭配方案
- 一键生成搭配出装图，发送通知提醒
- 采纳的搭配存入衣橱，形成个人穿搭档案

### 1.4 与 v0.1 方案的核心变化

| 维度 | v0.1（数字人+虚拟试穿） | v0.2（天气穿搭推荐） |
|------|--------------------------|----------------------|
| 核心功能 | 数字人生成 + AI 虚拟试穿 | 天气穿搭推荐 + 搭配出装图 |
| GPU 需求 | 6-8GB 显存 | 无 |
| 技术复杂度 | 高（SD+ControlNet+IDM-VTON） | 中（API 调用 + 图片拼接） |
| 硬件成本 | 1000-3000 元/月 | 0 |
| MVP 开发周期 | 17-27 天 | 10-15 天 |

## 2. 功能模块详细描述

### 2.1 模块一：用户注册与人身照上传

**功能描述**
用户首次登录后，上传一张全身清晰照，AI 分析体型和穿衣风格偏好。

**用户流程**
1. 微信授权登录（获取 openid）
2. 引导用户上传一张全身照（正面、光线充足、背景简洁）
3. 后端调用 MiMo-v2.5 分析照片，提取：体型特征、穿衣风格偏好
4. 分析结果存入用户档案，用于后续搭配推荐

**约束与规则**
- 照片要求：全身、正面、无遮挡、光线充足，尺寸不小于 500x500px
- 分析耗时约 3-5 秒
- 照片仅用于风格分析，不外传
- 支持重新上传更新风格档案

### 2.2 模块二：智能衣橱

**功能描述**
用户拍照上传衣服，系统自动完成：抠图去背景 -> 衣物识别分类 -> 存入衣柜。

**用户流程**
1. 点击「添加衣服」，调用相机或相册
2. 选择/拍摄多张照片（最多 9 张），图片立刻显示在列表中
3. **异步处理**（不阻塞用户）：
   - 每张图片后台上传到云存储
   - 写入数据库，状态标记为 pending
   - 调用 MiMo-v2.5 识别分类、颜色、材质、风格
   - 识别完成后更新数据库，状态标记为 recognized
   - 用户可实时看到每张图的处理状态（上传中→排队中→识别中→已识别）
4. 识别完成后，用户可点击任一图片编辑修改识别结果
5. 随时点击「完成」返回衣柜，不阻塞

**衣物分类体系**

| 一级分类 | 二级分类 | 说明 |
|----------|----------|------|
| 上衣 | 半袖/T恤 | 短袖类上衣 |
| 上衣 | 长袖衬衫 | 正式/休闲衬衫 |
| 上衣 | 卫衣/帽衫 | 连帽卫衣、圆领卫衣 |
| 上衣 | 毛衣/针织 | 毛衣、开衫、针织衫 |
| 上衣 | 背心/吊带 | 无袖类上衣 |
| 下装 | 长裤 | 牛仔裤、休闲裤、西裤、运动裤 |
| 下装 | 短裤 | 牛仔短裤、运动短裤、休闲短裤 |
| 下装 | 半裙 | 短裙、中长裙、百褶裙、A字裙 |
| 连衣裙 | 连衣裙 | 连衣裙（独立大类，非上下装组合） |
| 外套 | 薄外套 | 防晒衣、薄夹克、牛仔外套 |
| 外套 | 厚外套 | 羽绒服、棉服、毛呢大衣 |
| 外套 | 西装/正装 | 西装外套、小香风外套 |
| 鞋子 | 运动鞋 | 运动鞋、跑鞋、板鞋 |
| 鞋子 | 皮鞋/靴子 | 皮鞋、马丁靴、切尔西靴 |
| 鞋子 | 凉鞋/拖鞋 | 凉鞋、拖鞋、帆布鞋 |
| 配饰 | 包/帽/围巾 | 包包、帽子、围巾、手套 |

**搭配组合规则**
- 常规搭配：上衣 + 下装 + 鞋子（3 件套）
- 连衣裙搭配：连衣裙 + 鞋子（2 件套，无需上下装）
- 完整搭配：上衣 + 下装 + 鞋子 + 外套（4 件套，寒冷天气）
- 可选加配饰

**衣柜展示**
- 一级 Tab 筛选：全部 / 上衣 / 下装 / 连衣裙 / 外套 / 鞋子 / 配饰
- 二级筛选（可选）：点击一级分类后显示该分类下的二级标签
- 每张卡片显示：抠图后衣服图 + 二级分类标签 + 颜色标记
- 支持删除、编辑信息

### 2.3 模块三：AI 搭配推荐（核心模块）

**功能描述**
根据用户人身照（体型风格）、当天天气、衣柜中已有衣服，生成 2-3 套搭配方案。

**用户流程**
1. 首页展示今日天气卡片（温度/天气/穿衣建议摘要）—— **同步**
2. 展示 2-3 套推荐搭配方案 —— **异步**（首次加载需 5-10 秒，有加载动画）
3. 点击方案卡片进入搭配详情页 —— **同步**
4. 详情页显示：搭配出装图 + 详细搭配说明 + 采纳/舍弃按钮 —— **同步**
5. 点击「采纳」→ 生成衣橱照 → 存入穿搭档案 —— **异步**（后台处理，用户立刻看到成功提示）

**推荐逻辑**
1. 规则层（快速、零成本）：温度区间 + 天气状况 -> 匹配衣柜中合适的衣服类别
2. AI 层（MiMo-v2.5）：
   - 输入：用户体型风格 + 天气数据 + 衣柜衣服列表（含图片）
   - 输出：2-3 套搭配方案（JSON 结构化数据）
   - 每套方案包含：上衣/裤子/鞋子选择 + 搭配理由 + 适合场合

**搭配出装图生成**
- 方案：将选中的 3 件衣服（上衣+裤子+鞋子）的抠图透明底图，用 Pillow 拼接到一张白底/浅色底图上
- 布局：上衣居中上方，裤子居中中部，鞋子居中下方
- 输出尺寸：750x1000px（适配小程序展示）
- 速度：< 1 秒

**搭配数据结构（MiMo-v2.5 JSON 输出）**
`json
{
  "outfits": [
    {
      "name": "春日休闲通勤",
      "weather_context": "18度 多云微风",
      "items": {
        "top": {"item_id": "xxx", "name": "白色衬衫", "image_url": "..."},
        "bottom": {"item_id": "yyy", "name": "深蓝牛仔裤", "image_url": "..."},
        "shoes": {"item_id": "zzz", "name": "白色帆布鞋", "image_url": "..."}
      },
      "description": "今天18度多云微风，白色衬衫搭配牛仔裤是春季通勤经典组合...",
      "occasion": "通勤/日常",
      "score": 0.92
    }
  ]
}
`

### 2.4 模块四：通知提醒

**功能描述**
搭配方案生成后，通过多通道通知用户查看。

**通知方式**
| 方式 | 触发条件 | 内容 |
|------|----------|------|
| 微信订阅消息 | 用户授权后 | 「今日穿搭已准备好，点击查看」 |
| 站内信通知 | 默认开启 | 小程序内红点提示 |

**通知时机**
- 每日预生成：凌晨 2:00-5:00 为活跃用户预生成穿搭方案 —— **异步**（后台定时任务）
- 实时生成：用户手动触发时实时生成 —— **异步**（生成过程后台进行，完成后推送通知）
- 生成完成后立即推送站内信通知 —— **异步**

### 2.5 模块五：搭配采纳与衣橱管理

**功能描述**
用户采纳的搭配方案，将三件衣服合到一张衣橱照中，存入个人穿搭档案。

**用户流程**
1. 在搭配详情页点击「采纳」
2. 系统将三件衣服（上衣+裤子+鞋子）合成一张衣橱照
3. 搭配记录存入「我的穿搭」列表
4. 用户可查看历史穿搭记录

**衣橱照生成**
- 与搭配出装图相同的拼接方案
- 额外添加：搭配名称、日期水印
- 存储到云存储，关联用户穿搭记录

### 2.6 模块六：管理员配置（仅管理员可见）

**功能描述**
管理员手机号 ADMIN_PHONE 可在小程序内配置 MiMo 模型参数。

**可配置项**
| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 模型选择 | mimo-v2.5 | 可切换为 mimo-v2.5-pro |
| temperature | 0.7 | 生成多样性，范围 [0, 1.5] |
| top_p | 0.95 | 核采样阈值，范围 [0.01, 1.0] |
| 每日推荐套数 | 3 | 每次生成的搭配方案数 |
| 推荐 prompt | (预设) | 搭配推荐的系统提示词 |

**权限控制**
- 配置页面入口仅对管理员手机号可见
- 其他用户访问配置页返回 403

## 4. 页面原型概要

### 4.1 首页
- 顶部：天气卡片（温度/天气图标/穿衣建议摘要）

- 列表展示已采纳的搭配记录

## 8. 同步/异步流程说明

### 8.1 同步 vs 异步总览

| 动作 | 类型 | 耗时 | 用户感知 |
|------|------|------|----------|
| 微信登录 | 同步 | < 1s | 立刻跳转 |
| 天气数据获取 | 同步 | < 1s | 首页直接展示 |
| 衣服上传（拍照/选图） | 同步 | < 2s | 立刻出现在列表 |
| 衣服上传到云存储 | 异步 | 2-5s | 列表显示"上传中" |
| MiMo-v2.5 衣物识别 | 异步 | 3-8s | 列表显示"识别中" |
| 识别结果编辑保存 | 同步 | < 1s | 立刻更新 |
| 搭配推荐生成 | 异步 | 5-10s | 首页显示加载动画 |
| 出装图拼接 | 异步 | < 1s | 推荐完成后展示 |
| 搭配采纳 | 异步 | 2-3s | 提示"已采纳"，后台存档 |
| 站内信通知 | 异步 | < 1s | 红点提示 |
| 每日预生成搭配 | 异步 | 30-60s/人 | 凌晨后台执行，用户无感知 |

    │     {category, subcategory, color, material,     │

## 9. 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 搭配采纳率 | >= 60% | 用户采纳推荐搭配的比例 |
