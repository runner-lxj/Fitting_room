## 1. 系统架构

### 1.1 整体架构

`
+-------------------+     +-------------------+     +-------------------+
|   微信小程序       | --> |   微信云开发       | --> |   外部 API        |
|   (WXML+WXSS+JS)  |     |   (云函数+云数据库  |     |   MiMo-v2.5       |
|                    |     |    +云存储)        |     |   和风天气         |
+-------------------+     +-------------------+     |   remove.bg       |
                                                     +-------------------+
`

### 1.2 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | 微信小程序原生 | WXML + WXSS + JS |
| 业务后端 | 微信云开发 | 云函数 + 云数据库 + 云存储 |
| AI 推理 | MiMo-v2.5 API | OpenAI Chat Completions 兼容 |
| 衣服抠图 | rembg / remove.bg API | CPU 可运行 |
| 出装图拼接 | Pillow | Python 图片处理库 |
| 天气数据 | 和风天气 API v7 | 免费版 1000 次/天 |

| 分组键 | category | subcategory 范围 | 说明 |

**内搭/外搭的角色分配**：同一件衣服（如半袖）在不同搭配中可以扮演不同角色 — 单穿时 role 为 `top`，穿在外套里面时 role 为 `inner`。角色由大模型根据搭配场景决定，不在衣橱数据中预设。

**内搭/外搭的角色分配**：同一件衣服（如半袖）在不同搭配中可以扮演不同角色 — 单穿时 role 为 `top`，穿在外套里面时 role 为 `inner`。角色由大模型根据搭配场景决定，不在衣橱数据中预设。

```
输入：
  - user_id: 用户 openid
  - trigger: "manual" | "scheduled"

处理流程：
  1. 获取用户 style_profile（体型风格）
  2. 获取用户衣柜 clothes，按 6 类分组为 wardrobe 对象
  3. 调用和风天气 API 获取当前天气
  4. 构造 MiMo-v2.5 请求：
     - system: 搭配推荐 prompt（见 4.3 节 v0.3 版本）
     - user: Weather: {temp}C {condition} Wardrobe: {wardrobe_json}
     - response_format: json_object
  5. 解析 MiMo-v2.5 返回的搭配 JSON（items 为数组格式）
  6. 将 result.reason 映射到 ai_reason
  7. 校验 item id 是否存在于衣柜
  8. 保存搭配方案到 outfits 集合
  9. 异步触发出装图拼接
  10. 发送站内信通知

输出：
  - outfits: 搭配方案列表（含 ai_reason、style_summary）
```

#### outfit-compose（出装图拼接）

> **v0.3 变更**：items 从固定 3 件改为动态数量，按 role 排序布局。

```
输入：
  - outfit_id: 搭配方案 ID

处理流程：
  1. 从 outfits 集合获取搭配方案
  2. 遍历 items 数组（兼容旧 object 格式），获取每件衣服的 cutout_url
  3. 按 role 排序：outer -> inner -> top -> bottom -> shoes -> accessory
  4. Pillow 动态拼接：
     - 创建 750x1200 白色底图（高度根据件数动态调整）
     - 从上到下依次排列，每件高度 = 画布高度 / 总件数
     - 宽度统一 55%，居中
  5. 上传到云存储
  6. 更新 outfits 集合的 composition_url

输出：
  - composition_url: 出装图云存储 URL
```

### 4.1 API 调用封装

`python
# 云函数中调用 MiMo-v2.5
import json
from openai import OpenAI

def call_mimo(messages, config):
    client = OpenAI(
        api_key=config["api_key"],
        base_url="https://api.xiaomimimo.com/v1"
    )
    
    completion = client.chat.completions.create(
        model=config.get("model", "mimo-v2.5"),
        messages=messages,
        response_format={"type": "json_object"},
        temperature=config.get("temperature", 0.7),
        top_p=config.get("top_p", 0.95),
        max_completion_tokens=4096
    )
    
    return json.loads(completion.choices[0].message.content)
`

### 4.2 图片输入方式

**云存储图片 -> MiMo-v2.5 的流程**：
1. 云存储中的图片需要先获取临时访问 URL
2. 将临时 URL 传入 MiMo-v2.5 的 image_url
3. 或者下载图片后转为 Base64 传入

**推荐方案**：使用 Base64 编码（更可靠，不依赖临时 URL 有效期）

`python
import base64

def image_to_base64_url(cloud_file_id):
    # 从云存储下载图片
    file_content = cloud.get_file(cloud_file_id)
    b64 = base64.b64encode(file_content).decode()
    mime = "image/jpeg"  # 根据实际格式
    return f"data:{mime};base64,{b64}"
`

#### v0.3 Prompt（新版，多件灵活组合）

```
你是专业的穿搭推荐助手。根据用户的衣橱和当前天气，推荐 3 套搭配。

## 衣橱数据结构
- tops: 上衣
  - 半袖/T恤：适合单穿或做内搭
  - 长袖T恤/POLO衫：通勤休闲皆可
  - 衬衫（长袖/短袖）：可单穿，短袖衬衫也可做外搭
  - 雪纺衫/针织衫：女性通勤常见上衣
  - 卫衣/毛衣：秋冬主力
  - 背心/吊带：夏天内搭或单穿
- bottoms: 下装
  - 长裤：牛仔裤、阔腿裤、西裤、直筒裤、烟管裤
  - 短裤：牛仔短裤、运动短裤、西装短裤
  - 半裙：A字裙、百褶裙、包臀裙、伞裙（分短裙/中长裙/长裙）
  - 打底裤：秋冬内搭
- dresses: 连衣裙
  - 连衣裙：短款/中长款/长款，可单独成套
  - 背带裙：可搭配内搭上衣
- outers: 外搭/叠穿层
  - 薄外套：开衫、棒球服、牛仔外套
  - 厚外套：风衣、棉服、羽绒服
  - 西装：通勤必备
- shoes: 鞋子
  - 运动鞋/帆布鞋：休闲百搭
  - 皮鞋/乐福鞋：通勤正式
  - 高跟鞋/平底鞋/单鞋：女性日常
  - 短靴/长靴：秋冬搭配
  - 凉鞋/拖鞋：夏季
- accessories: 配饰
  - 包、帽、围巾/丝巾、腰带、首饰

## 穿搭规则
1. 每套搭配 2~5 件单品，灵活组合，不要死板地只做"上衣+裤子"。
2. 善用叠穿：一件半袖可以单穿做 top，也可以穿在外套里面做 inner。
   外套、开衫、不系扣的衬衫适合做 outer，营造层次感。
3. 有鞋子就搭配鞋子，没有就省略。
4. 连衣裙可以单独成套，也可以搭配 outer 和 shoes。
5. 根据温度判断穿搭厚度：
   - 30C+：轻薄透气，短袖短裤凉鞋为主
   - 25~30C：可以薄外套+短袖叠穿
   - 18~25C：卫衣/薄外套/长裤
   - 10~18C：厚外套/毛衣/长裤
   - 10C 以下：保暖为主
6. 搭配要有发散思维，不要每次都推荐相同的组合。
   考虑色彩对比、材质混搭、风格碰撞（如休闲+运动、简约+知性）。
7. 女装搭配更注重细节：腰线、裙长、配饰点缀、内外层次的比例。
8. 只能使用衣橱中已有的 id，绝对不要编造 id。

## 输出格式（严格 JSON）
{
  "outfits": [
    {
      "name": "搭配名称（4-8字，有画面感）",
      "occasion": "场合（如：日常出行/通勤/约会/运动）",
      "score": 0.85,
      "style_summary": "风格关键词（如：简约清爽/知性通勤/休闲运动）",
      "ai_reason": "搭配理由（2-4句话，说明为什么这样搭好看、颜色搭配逻辑、适合的场景、整体风格感受）",
      "items": [
        { "id": "衣服id", "role": "inner" },
        { "id": "衣服id", "role": "outer" },
        { "id": "衣服id", "role": "bottom" },
        { "id": "衣服id", "role": "shoes" }
      ]
    }
  ]
}

## role 取值
- top: 单穿上衣（不分内外）
- inner: 内搭（穿在外套里面的那件）
- outer: 外搭（叠穿在最外层，如开衫、薄外套、不系扣衬衫）
- bottom: 下装
- shoes: 鞋子
- accessory: 配饰

只输出 JSON，不要其他文字。
```

### 6.1 管理员权限控制

`javascript
// 云函数中验证管理员身份
function checkAdmin(openid) {
  const user = db.collection('users').doc(openid).get();
  return user.phone === 'ADMIN_PHONE';
}

// 小程序端：隐藏管理员入口
// 仅当 openid 对应用户为管理员时显示配置入口
`

### 6.2 数据安全

- 用户照片仅用于 AI 分析，不外传、不用于其他用途
- MiMo-v2.5 API 调用使用服务端 API Key，不暴露给前端
- 云数据库规则：用户只能读写自己的数据
- 所有传输使用 HTTPS

## 穿搭规则
1. 每套搭配 2~5 件单品，灵活组合，不要死板地只做"上衣+裤子"。
2. 善用叠穿：一件半袖可以单穿做 top，也可以穿在外套里面做 inner。
   外套、开衫、不系扣的衬衫适合做 outer，营造层次感。
3. 有鞋子就搭配鞋子，没有就省略。
4. 连衣裙可以单独成套，也可以搭配 outer 和 shoes。
5. 分析每件衣服的材质、厚度、款式，判断适合什么天气穿：
   - 轻薄透气材质（棉麻、雪纺）→ 适合炎热天气
   - 适中厚度（针织、牛仔）→ 适合凉爽天气
   - 厚实保暖材质（毛呢、羽绒）→ 适合寒冷天气
   - 防水材质 → 适合雨天
6. 搭配要有发散思维，不要每次都推荐相同的组合。
   考虑色彩对比、材质混搭、风格碰撞（如休闲+运动、简约+知性）。
7. 女装搭配更注重细节：腰线、裙长、配饰点缀、内外层次的比例。
8. 只能使用衣橱中已有的 id，绝对不要编造 id。
9. 不同方案覆盖不同天气场景（炎热、凉爽、寒冷、雨天等）。
