# 智能穿搭助手 — 技术可行性分析

> 版本: v0.2 | 更新日期: 2026-07-07 | 状态: 新方向定稿

## 1. 技术约束条件

| 约束项           | 说明                                                          |
| ---------------- | ------------------------------------------------------------- |
| 前端             | 微信小程序（WXML + WXSS + JS），不使用 UniApp/Taro            |
| 业务后端         | 微信云开发（云函数 + 云数据库 + 云存储），无自建服务器        |
| AI 推理          | MiMo-v2.5 API（OpenAI Chat Completions 兼容格式）            |
| 衣服抠图         | rembg（U2-Net），CPU 可跑                                     |
| 搭配出装图       | Pillow 图片拼接，零成本                                       |
| 预算             | MVP 阶段零硬件成本，全部使用 API 和开源工具                   |

---

## 2. 架构总览

`
微信小程序 (前端)
    |
    v
微信云开发 (业务层：云函数 + 云数据库 + 云存储)
    |
    +---> MiMo-v2.5 API (AI 搭配推荐 + 衣物识别 + 风格分析)
    +---> 和风天气 API (天气数据)
    +---> GPU 推理服务器 (仅衣服抠图，可选)
`

关键决策：**彻底去掉 GPU 依赖**。所有 AI 功能通过 MiMo-v2.5 API 完成，衣服抠图用 rembg（CPU 可运行）。

## 3. 各模块技术选型

### 3.1 MiMo-v2.5 API 调用

**确认信息**
- API 地址：https://api.xiaomimimo.com/v1/chat/completions
- 认证方式：pi-key header 或 Authorization: Bearer token
- 模型 ID：mimo-v2.5（全模态理解，支持图片输入）
- 兼容格式：OpenAI Chat Completions API
- 上下文窗口：1M tokens，最大输出 128K tokens
- 速率限制：RPM 100，TPM 10M

**核心能力**
| 能力 | 支持情况 | 用途 |
|------|----------|------|
| 图片理解 | ✅ 支持 URL 和 Base64 | 分析人身照、识别衣物 |
| 结构化输出 | ✅ response_format: json_object | 搭配方案 JSON 输出 |
| 工具调用 | ✅ 支持 | 可选扩展 |
| 流式输出 | ✅ 支持 | 搭配生成进度展示 |
| 深度思考 | ✅ 支持 | 复杂搭配推理 |

**调用示例（Python）**
`python
from openai import OpenAI

client = OpenAI(
    api_key="your-mimo-api-key",
    base_url="https://api.xiaomimimo.com/v1"
)

completion = client.chat.completions.create(
    model="mimo-v2.5",
    messages=[
        {"role": "system", "content": "你是穿搭推荐专家..."},
        {"role": "user", "content": [
            {"type": "image_url", "image_url": {"url": "用户人身照URL"}},
            {"type": "text", "text": "根据天气和衣柜推荐搭配"}
        ]}
    ],
    response_format={"type": "json_object"},
    temperature=0.7,
    top_p=0.95
)
`

**超参数**
| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| temperature | 1.0 | [0, 1.5] | 思考模式下强制 1.0 |
| top_p | 0.95 | [0.01, 1.0] | 思考模式下强制 0.95 |

**注意**：思考模式下 temperature 和 top_p 不可自定义，会被强制设为 1.0 和 0.95。

### 3.2 衣服抠图

推荐方案：rembg（基于 U2-Net / RMBG-2.0）

| 方案                 | 效果 | 成本 | 速度     | GPU |
| -------------------- | ---- | ---- | -------- | --- |
| rembg (U2-Net/RMBG) | 优秀 | 免费 | 0.5-1s   | 可选 |
| SAM (Segment Anything)| 优秀 | 免费 | 2-4s   | 需要 |
| remove.bg API        | 优秀 | 付费 | 0.3s     | 不需要 |

**选择 rembg 的理由**：
- 本地之前已验证过 OpenCV GrabCut 作为备选
- CPU 可运行，不需要 GPU
- 生产环境（正常 Windows/Linux）可正常运行
- 微信云函数内存限制 256MB，rembg 模型约 40MB，可行

**云函数适配方案**：
- 方案 A（推荐）：部署独立抠图服务（轻量级 Flask/FastAPI），云函数调用
- 方案 B：在云函数中直接运行 rembg（需确认内存是否足够）
- 方案 C：使用 remove.bg 免费 API（50 张/月免费）

### 3.3 搭配出装图生成

  |     裤子（居中中部 40%）         |

| 维度 | 指标 |

| 底图样式 | 纯白 / 浅灰渐变 / 品牌色底图（可配置） |

## 4. MiMo-v2.5 数据解析验证

### 4.1 结构化输出能力

MiMo-v2.5 支持 esponse_format: {"type": "json_object"}，确保输出为合法 JSON。

**关键点**：
- json_object 模式仅保证 JSON 语法合法，不保证字段结构
- 需要在 system prompt 中明确指定 JSON 结构模板
- 建议使用 jsonschema 库做后验校验
- 失败时可重试或降级

### 4.2 图片理解能力

MiMo-v2.5 支持多图输入，可同时分析：
- 用户人身照（体型、风格）
- 衣柜衣服图片（类别、颜色、材质）

**输入限制**：
- 图片格式：JPEG, PNG, GIF, WebP, BMP
- 单张图片大小：URL 方式 < 50MB，Base64 方式 < 50MB
- 多图总 Token 不超过上下文窗口

### 4.3 搭配推荐 Prompt 设计

`
系统提示词（管理员可配置）：
你是专业的穿搭推荐顾问。根据用户的体型风格、当前天气和衣柜中的衣服（含二级分类如半袖、长裤、连衣裙等），
推荐 2-3 套搭配方案。

输出要求（JSON 格式）：
{
  "outfits": [
    {
      "name": "方案名称",
      "weather_context": "天气描述",
      "items": {
        "top": {"item_id": "衣柜衣服ID", "name": "衣服名称"},
        "bottom": {"item_id": "衣柜衣服ID", "name": "衣服名称"},
        "shoes": {"item_id": "衣柜衣服ID", "name": "衣服名称"}
      },
      "description": "详细搭配说明（50-100字）",
      "occasion": "适合场合",
      "score": 0.0-1.0
    }
  ]
}

规则：
- 常规搭配：上衣+下装+鞋子；连衣裙搭配：连衣裙+鞋子；寒冷天气可加外套
- 只能从用户衣柜中选择
- 搭配说明要具体，说明为什么这样搭
- 考虑颜色搭配、风格协调、场合适配
`

## 5. GPU 服务器需求

### 结论：不需要 GPU 服务器

| 模块 | 之前的方案 | 新方案 |
|------|------------|--------|
| 数字人生成 | SD + ControlNet (4GB) | 不需要 |
| 虚拟试穿 | IDM-VTON (4GB) | 不需要 |
| 衣服抠图 | rembg (0.5GB) | rembg (0.5GB) 或 API |

**总显存需求：0 GB**（抠图可用 CPU，或用 remove.bg API）

## 6. 微信云开发适配

### 能做的（云函数）
- 用户数据 CRUD（衣柜、个人信息、搭配历史）
- 天气 API 调用
- MiMo-v2.5 API 调用
- Pillow 图片拼接（需确认内存）
- 文件上传/下载管理
- 定时任务触发（凌晨预生成）

### 需要外部服务的
- 衣服抠图：rembg 需要 onnxruntime，云函数可能受限
  - 备选：部署独立抠图服务，或使用 remove.bg API

### 云函数调用流程
`
用户触发推荐 -> 云函数接收请求
    -> 获取用户衣柜数据 + 天气数据
    -> 调用 MiMo-v2.5 API（传入图片 URL + 衣柜列表）
    -> 解析 JSON 响应
    -> 用 Pillow 拼接搭配出装图
    -> 上传到云存储
    -> 返回给小程序
    -> 发送站内信通知
`

### 延迟预估
| 操作             | 总耗时    | 构成                      |
| ---------------- | --------- | ------------------------- |
| 衣服入库（抠图） | 3-5s      | rembg 1-3s + 分类 0.5s   |
| 搭配推荐         | 5-10s     | MiMo-v2.5 API 3-8s       |
| 出装图拼接       | < 1s      | Pillow 拼接               |
| 人身照分析       | 3-5s      | MiMo-v2.5 API 3-5s       |

## 10. 结论

**技术可行性：完全可行，且显著优于原方案。**

1. MiMo-v2.5 做搭配决策和文案生成：**完全可行**。OpenAI 兼容格式，支持图片输入 + JSON 结构化输出，1M 上下文窗口足够承载衣柜数据。
2. 搭配出装图拼接：**完全可行**。Pillow 零成本，< 1 秒，效果不差。
3. 衣服抠图：**可行**。rembg CPU 可跑，或用 remove.bg API。
4. 天气穿搭推荐：**可行**。规则引擎零成本 + MiMo-v2.5 精细推荐。
5. 全链路零 GPU 依赖：**已确认**。开发和运维成本大幅降低。

**与原方案对比**：去掉数字人和虚拟试穿后，技术风险从「高」降为「低」，开发周期缩短 40%，硬件成本归零。核心用户体验从「虚拟试穿」变为「智能搭配推荐+出装图」，实用性更强。
