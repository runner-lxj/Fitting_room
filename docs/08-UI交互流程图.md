# 穿搭助手小程序 — UI 交互流程图

> V1.0 | 2026-07-13

## 1. 用户旅程

首次打开 -> 隐私协议 -> 自动登录(随机昵称+灵悦默认头像) -> 首页 -> 添加衣服 -> 一键搭配 -> 查看/采纳/舍弃

## 2. 首页流程

加载 -> 获取天气 + 衣柜/穿搭计数 -> 从accepted按温度排序获取推荐 -> 展示或空状态
操作: 点击推荐->详情页, 换一批->切换, 快捷入口->跳转

## 3. 一键搭配(核心)

点击 -> 按钮loading+浮窗弹出 -> 调用outfit-recommend -> MiMo生成 -> 完成
浮窗状态: generating(进度条+脉冲点) -> done(绿点+查看方案)
按钮状态: 始终显示"一键搭配"

## 4. 搭配采纳

点击卡片 -> 详情页 -> 采纳(status=accepted,match+1) / 舍弃(status=rejected,返回列表)
批量模式: tab栏右侧圆图标进入 -> 多选 -> 底栏全选/采纳N套 -> 确认弹窗

## 5. 衣服入库

FAB+ -> chooseMedia -> uploading -> 云存储+数据库 -> 识别 -> recognized/failed
编辑: 点击已识别项 -> 修改分类/颜色 -> 保存

## 6. 页面缓存机制

脏标记(app.globalData):
- outfitDirty: 采纳/舍弃后true, my-outfits onShow消费
- wardrobeDirty: 上传/删除后true, wardrobe onShow消费
首次进入(_firstShow=true)必定加载, 后续仅脏时刷新

## 7. 方案去重

穿搭列表按衣物ID集合去重: items.map(i=>i.id).sort().join(',') 作为key
tab计数基于去重后数量

## 8. 状态管理

衣服: uploading -> pending -> recognizing -> recognized/failed
方案: pending -> accepted/rejected
计数: match_count = accepted outfits中包含该衣服的数量

## 9. 页面跳转

TabBar: 首页/衣柜/穿搭/我的
子页面: 添加衣服(衣柜FAB), 搭配详情(穿搭列表), 管理员(我的)
返回: navigateBack
